# 鳴潮矩陣編隊工具 (DPM Builder) - 版本更新歷史

本專案的所有顯著更改、功能新增與錯誤修復，都將記錄於此文件中。

### 🏷️ 關於版本命名規範 (Versioning Convention)
本專案正式採用 **[語意化版本控制 (Semantic Versioning)](https://semver.org/lang/zh-TW/)** 規範，格式為 `主版號.次版號.修訂號 (Major.Minor.Patch)`：
* **Major (主版號)**：進行不相容的 API 修改、底層架構大翻新。
* **Minor (次版號)**：以向下相容的方式新增功能。
* **Patch (修訂號)**：以向下相容的方式修復 Bug，或純粹進行視覺/數值更新。

## [4.7.6] - 2026-03-19
### Added (新增)
- **🧠 束式搜索 (Beam Search) 編隊引擎**：取代原先的貪婪演算法。系統在運算時會保留多組不衝突的分支，以計算出整體 16 隊的最高總分。
- **📊 戰前分析與時間預估**：執行編隊前，系統會依據疲勞衝突率估算有效組合數，並提供全域搜索與預設深度的運算時間預估，允許使用者自訂搜尋深度 (`beamWidth`)。
- **🤖 資料庫自動填補**：實戰洗牌完成後，提供自動填補選項。系統會結算已上場角色的疲勞值，並從資料庫依序挑選基礎 DPS 最高且不衝突的排軸填滿空位。

### Changed (優化與調整)
- **🌍 全局抗性環境設定**：將抗性遭遇選項從單一隊伍獨立欄位中抽離，移至「環境設定」面板作為全局變數，使運算邏輯更符合矩陣模式一體適用的環境規則。
- **🔄 反推邏輯與動態校正**：實戰反推時會先排除隊伍自訂加權 (Buff)，計算出純粹基礎 DPS，並與理論區間比對：高於上限則更新基礎 DPS 並重置穩定度；低於下限則保留基礎 DPS，僅下修穩定度。
- **公平排序基準**：洗牌時統一採用當次實戰打出的絕對真實火力 (`trueBaseDps`) 作為排序依據，修正舊版雙重扣除穩定度的誤差。
- **文本客觀化**：全面修正操作指南與介面提示，移除誇飾性用語，確保系統行為描述平實精確。

## [4.7.0] - 2026-03-16
