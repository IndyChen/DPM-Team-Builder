# 鳴潮矩陣疊兵編隊工具 (DPM Builder) - 版本更新歷史

本專案的所有顯著更改、功能新增與錯誤修復，都將記錄於此文件中。

### 🏷️ 關於版本命名規範 (Versioning Convention)
本專案正式採用 **[語意化版本控制 (Semantic Versioning)](https://semver.org/lang/zh-TW/)** 規範，格式為 `主版號.次版號.修訂號 (Major.Minor.Patch)`：
* **Major (主版號)**：進行不相容的 API 修改、底層架構大翻新。
* **Minor (次版號)**：以向下相容的方式新增功能。
* **Patch (修訂號)**：以向下相容的方式修復 Bug，或純粹進行數值更新。

---
## [v4.6.3] - 2026-03-14 15:30

🌟 **嚴格配隊邏輯與資料庫重構 (Strict Logic & DB Refactoring)**

* **資料庫架構大升級**：將底層 `dpsData` 陣列全面重構為以「主輸出」為節點的樹狀結構 (`teamDB`)，大幅降低後續維護與新增隊伍的難度。
* **嚴格配隊推薦引擎**：重寫推薦邏輯。現在推薦的 2、3 號位必須能與當前已選的隊友組成資料庫內真實存在的隊伍，徹底杜絕同隊出現 DPS 數值打架的「縫合怪」無效組合。
* **動態 DPS 預測**：未滿編 3 人的隊伍，下拉選單僅顯示星號推薦而不顯示 DPS；系統會自動過濾因隊友耗盡而無法組建的排軸，確保選單中顯示的 DPS 為「當下真實可組建」的最高預期值。
* **防彈級重整修復**：強化 LocalStorage 的讀取與驗證邏輯，遭遇不相容或遺失的舊資料會自動淨化重置，徹底解決網頁重整後白畫面、找不到角色或排軸消失的嚴重 Bug。
* **耐力與生存位雙置頂**：左側「剩餘可用次數」面板恢復了「生存位 / 一般角色」分類，並將仍有剩餘次數的角色永遠置頂；3號位（生存/輔助）選單的「其他角色」區塊亦會將所有生存位強制排序於最上方。
* **一鍵按鈕狀態同步**：修復因篩選隱藏角色導致「一鍵全選/清空」按鈕失效的邏輯衝突，紅綠按鈕狀態現在可完美動態切換並完全支援重整記憶。
* **篩選視覺強化**：上方過濾按鈕（星級/版本）在未選取（隱藏）狀態下，改為明顯的暗灰色對比，提供更直覺的視覺回饋。
* **光主與 1 號位推薦異常修復**：修正選單渲染邏輯，防止在特定操作順序下導致光主選項被異常清除；並修正誤將純輔助黑名單套用於所有位置的錯誤，確保 1 號位在單獨替換時能正確顯示推薦清單。
* **專有名詞校正**：全域介面與提示訊息中的「排班」統一修正為更符合遊戲語境的「編隊」。
* **排軸資料庫還原**：補回先前版本遺失的完整 80+ 組 Meta 與非主流隊伍資料，並修復了難度過濾標籤（如：🟩 輪椅）無法作動的問題。

## [v4.6.2] - 2026-03-14 01:30
### 🌟 全面優化與校正 (Optimization & Fixes)
- **重整 UI 修復**：修復了網頁重新整理後，「清空/全選排軸」按鈕未能正確讀取 LocalStorage 狀態，導致全空時按鈕仍顯示紅色的 Bug。
- **排軸快搜實裝**：在「配置實戰排軸」區塊新增專屬搜尋框，支援簡繁體與部分匹配（如輸入簡體「卡卡罗」也能找到），找特定角色更有效率。
- **排軸快篩與人因優化**：新增 4 顆特定難度（🟩 / 🔵 / ⭐ / ⚠️）的常駐獨立開關。將按鈕群組化並將總開關固定於首位，縮短滑鼠移動距離（符合費茨法則）。
- **數據全面校正**：依照最新社群計算數據，全面更新卡卡羅、菲比、愛彌斯等所有隊伍的排軸與 DPS 數值。
- **漂泊者型態分離系統**：配隊選單正式將漂泊者分離為「光主、暗主、風主」。三者完美共用於底層的「漂泊者」耐力上限，杜絕超用 Bug。
