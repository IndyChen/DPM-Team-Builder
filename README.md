專為《鳴潮》(Wuthering Waves)「矩陣疊兵」模式設計的純前端開源排班與 DPS 計算輔助工具。幫助玩家在深厚的角色池中，精準分配耐力、過濾實戰排軸，並輕鬆完成極限 16 隊的最佳化編排！

🌐 **[https://indychen.github.io/DPM-Team-Builder/]**

---

## ✨ 核心特色 (Features)

### 🚀 純前端極致架構 (Zero-Backend)
- **零伺服器延遲**：100% 運行於瀏覽器端，過濾與計算瞬間完成，無懼高併發卡頓。
- **雲端無縫存檔**：採用 `LocalStorage` 技術，關閉網頁後自動保留您的「角色背包」、「排軸勾選」與「排班進度」，注重隱私且極度便利。

### 📊 動態排軸與 DPS 數據庫
- 內建最新 **3.1 版本** 主流配隊與 DPS 期望值。
- **難度快篩系統**：根據自身操作能力，一鍵開關特定難度的排軸（🟩 輪椅 / 🔵 中等 / ⭐ 進階 / ⚠️ 極難）。系統將自動連動下拉選單，只推薦您「會打且能打出最高傷害」的陣容。

### ⚔️ 智能選角與耐力追蹤
- **多型態角色系統**：完美支援「漂泊者」型態分離顯示（光主 / 暗主 / 風主），並於底層嚴格共用同一份耐力值，杜絕超用 Bug！
- **實時防呆機制**：嚴格還原矩陣疊兵機制（生存位可用 2 次，其餘角色 1 次）。當排班透支時，面板與選單將立即亮起紅色警告。
- **無縫搜尋引擎**：支援「簡繁體自動轉換」與「部分字串匹配」，在海量角色與排軸中一秒精準定位。

### 📷 一鍵長圖匯出
- 內建 `html2canvas` 渲染引擎，排班完成後只需點擊「匯出圖片」，系統會自動過濾空白關卡，生成精美的深色電競風實戰排班表，方便於社群/論壇分享。

### 🛠️ 樹狀資料庫設計 (維護者福音)
- 底層資料庫 (`teamDB`) 採用以主輸出 (Main DPS) 為節點的樹狀結構，未來更新版本、新增隊伍排軸極度直觀且不易出錯。

## 🛠️ 使用說明 (How to Use)

本專案無需安裝任何 Node.js 或後端環境，**開箱即用**：
1. 下載專案原始碼，或直接透過瀏覽器開啟 `index.html`。
2. **第一步 [勾選擁有角色]**：建立您的專屬角色池，支援中文搜尋。
3. **第二步 [配置實戰排軸]**：剔除您不想打的極難或冒煙軸。
4. **第三步 [隊伍編排]**：利用「世代隊伍快速填入」或手動下拉選單進行排班，系統會自動將最高 DPS 的適配角色置頂。
5. 完成後點擊 **[📷 匯出完成隊伍圖片]** 即可保存排班結果！

## 🚀 未來開發目標 (Future Roadmap)

我的願景是將本工具從「編隊規劃」進一步進化為**「矩陣分數模擬器」**。目前尚未實作，但預計推進的功能包含：

- **自訂真實面板導入**：允許玩家輸入或套用自身角色的真實裝備與面板數據，不再僅依賴理論期望值。
- **動態場域增益 (Buff) 系統**：實裝並套用「當期矩陣疊兵」的專屬場域 Buff 與環境特性。
- **極限分數精算引擎**：綜合玩家**「勾選的特定排軸」 × 「真實角色面板」 × 「當期場域 Buff」**，進行深度的隊伍流程模擬，直接預測並計算出該陣容的「理論極限通關分數」。
